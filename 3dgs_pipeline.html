一、数据获取

从 3DGS 官方仓库 下载 tandt_db.zip（682MB），只提取 Truck 原始照片，丢弃预计算的 COLMAP 结果。

# 下载
wget https://repo-sam.inria.fr/fungraph/3d-gaussian-splatting/datasets/input/tandt_db.zip

# 提取 Truck 照片
mkdir -p truck_data/input
unzip -q tandt_db.zip 'tandt/truck/images/*'
mv tandt/truck/images/* truck_data/input/

二、COLMAP SfM — 从照片到点云

2.1 特征提取

每张图提取 3000-7000 个 SIFT 特征。耗时 ~1.6 分钟。

export QT_QPA_PLATFORM=offscreen
mkdir -p truck_data/distorted/sparse

colmap feature_extractor \
    --database_path truck_data/distorted/database.db \
    --image_path truck_data/input \
    --ImageReader.single_camera 1 \
    --ImageReader.camera_model OPENCV \
    --SiftExtraction.use_gpu 0
⚠️ 关键优化: 251 张图穷举匹配需数小时。因为是视频帧序列，改用 sequential_matcher 只匹配相邻帧，仅需 ~4.6 分钟。

2.2 顺序特征匹配

colmap sequential_matcher \
    --database_path truck_data/distorted/database.db \
    --SiftMatching.use_gpu 0

2.3 稀疏重建（Bundle Adjustment）

colmap mapper \
    --database_path truck_data/distorted/database.db \
    --image_path truck_data/input \
    --output_path truck_data/distorted/sparse \
    --Mapper.ba_global_function_tolerance=0.000001
注册图像251 / 251
生成点云136,029 个
重投影误差0.34 px

2.4 畸变校正

colmap image_undistorter \
    --image_path truck_data/input \
    --input_path truck_data/distorted/sparse/0 \
    --output_path truck_data --output_type COLMAP

# 修正目录结构（3DGS 要求 sparse/0/）
mkdir -p truck_data/sparse/0
mv truck_data/sparse/*.bin truck_data/sparse/0/

最终数据集结构:

truck_data/
├── input/            ← 251 张原始 JPG
├── images/           ← 校正后的照片
├── distorted/        ← COLMAP 临时文件
└── sparse/0/
    ├── cameras.bin   ← 相机内参
    ├── images.bin    ← 每张图的位姿
    └── points3D.bin  ← SfM 点云 (136K 个点)

三、3DGS 训练

python train.py -s /path/to/truck_data -m output/truck --disable_viewer
参数值说明
-struck_data数据集路径
-moutput/truck模型输出路径
--disable_viewer-禁用 GUI 网络查看器
--eval不传全部 251 张用于训练

训练过程

  1. 从 SfM 点云初始化 136,029 个 3D 高斯
  2. 每步随机挑 1 张训练图 → 可微渲染 → L1 + SSIM 损失
  3. 反向传播更新 7 组参数：位置、SH DC、SH 高阶、缩放、旋转、不透明度、曝光
  4. 迭代 500–15,000: 自适应密度控制（克隆大梯度高斯、分裂大尺度高斯、剪枝近透明高斯）
  5. 每 3,000 步: 重置不透明度
  6. 每 1,000 步: 提升 SH 阶数（0 → 1 → 2 → 3）

四、渲染

python render.py -m output/truck --skip_test
  • 遍历 251 个训练视角，每个用完整 tile-based CUDA rasterizer 渲染
  • 使用 SH 全阶颜色（含视角依赖的高光效果）
  • 输出: train/ours_30000/renders/*.png + gt/*.png
  • 耗时: ~2 分钟
