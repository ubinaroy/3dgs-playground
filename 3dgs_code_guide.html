核心思想：用大量各向异性 3D 高斯椭球体表示场景。每个高斯有 59 个可优化参数（位置3 + 颜色48(SH) + 协方差7 + 不透明度1）。 通过可微分光栅化将高斯投影到 2D，与真实图像对比计算 loss，反向传播更新参数，并动态增删高斯（densification/pruning）。 论文: "3D Gaussian Splatting for Real-Time Radiance Field Rendering", Kerbl et al., SIGGRAPH 2023.
数据流：COLMAP SfM → 稀疏点云 (.ply) + 相机位姿 → Scene 加载 → GaussianModel 初始化 → training() 循环 { 随机选视角 → render() 光栅化 → 计算 loss → backward → optimizer.step → densify/prune } → 保存 .ply → render.py 出图 → metrics.py 算指标。

train.py — 训练主循环 (核心入口)

入口 training()
完整训练循环: 初始化 → 迭代 {取视角,渲染,算loss,backward,step,densify} → 保存/评测。约 30,000 次迭代。
辅助 prepare_output_and_logger()
创建输出目录、TensorBoard writer。
辅助 training_report()
记录 TensorBoard scalars + 在 test_iterations 时渲染 test/train 视图，计算 L1/PSNR。
# ── train.py: 核心训练循环 ──

def training(dataset, opt, pipe, testing_iterations,
              saving_iterations, checkpoint_iterations,
              checkpoint, debug_from):

    # 1. 初始化
    gaussians = GaussianModel(dataset.sh_degree)
    scene = Scene(dataset, gaussians)
    gaussians.training_setup(opt)
    if checkpoint:
        gaussians.restore(checkpoint)

    bg_color = [1,1,1] if dataset.white_background
               else [0,0,0]
    background = torch.tensor(bg_color, device="cuda")

    viewpoint_stack = scene.getTrainCameras().copy()
    viewpoint_indices = list(range(len(viewpoint_stack)))

    for iteration in range(1, opt.iterations + 1):

        # 2. 每 1000 步升一级 SH
        if iteration % 1000 == 0:
            gaussians.oneupSHdegree()

        # 3. 随机取一个训练视角 (不放回)
        rand_idx = randint(0, len(viewpoint_indices)-1)
        viewpoint_cam = viewpoint_stack.pop(rand_idx)
        if not viewpoint_stack:
            viewpoint_stack = scene.getTrainCameras().copy()

        # 4. 可微分光栅化渲染
        render_pkg = render(viewpoint_cam, gaussians, pipe, bg)
        image = render_pkg["render"]

        # 5. Loss = (1-λ)*L1 + λ*(1-SSIM)
        Ll1 = l1_loss(image, gt_image)
        ssim_val = ssim(image, gt_image)
        loss = (1.0 - opt.lambda_dssim) * Ll1 \
             + opt.lambda_dssim * (1.0 - ssim_val)

        # 6. 可选深度正则化
        if depth_l1_weight(iteration) > 0:
            loss += depth_l1_weight(it) * |invDepth - mono_invDepth|

        # 7. 反向传播
        loss.backward()

        with torch.no_grad():
            # 8. Densification (前15k步)
            if iteration < opt.densify_until_iter:
                gaussians.add_densification_stats(...)
                if iteration % opt.densification_interval == 0:
                    gaussians.densify_and_prune(...)
                if iteration % opt.opacity_reset_interval == 0:
                    gaussians.reset_opacity()

            # 9. Optimizer step
            gaussians.optimizer.step()
            gaussians.optimizer.zero_grad()

            # 10. 定期报告 & 保存
            if iteration in testing_iterations:
                training_report(...)
            if iteration in saving_iterations:
                scene.save(iteration)

training() — 主循环详解

这是整个项目的核心入口。输入是解析好的三类参数：ModelParams（数据路径）、OptimizationParams（训练超参）、PipelineParams（渲染管线选项）。

第1步 — 初始化：

  • GaussianModel(sh_degree) 创建一个空的 3D 高斯容器
  • Scene(dataset, gaussians) 读取 COLMAP 数据，从稀疏点云初始化高斯位置和颜色
  • training_setup(opt) 为各参数组创建 Adam optimizer，设置学习率 scheduler

第3步 — 随机视角采样：不放回地从训练集中 pop 一个相机，当栈空时重新填充。这确保了梯度计算的随机性，同时每个 epoch 内所有视角都被用到。

第4-7步 — 渲染+反向传播：核心计算——调用 CUDA rasterizer 投影高斯到 2D，合成图像，与 GT 比较。loss = (1-λ)·L1 + λ·(1-SSIM)，λ=0.2。L1 保留高频细节，SSIM 衡量结构相似性。

第8步 — 高斯致密化：仅在前 15k 步执行。基于 view-space 梯度累积判断高斯的"欠重建"程度：梯度大的分裂 (split)，梯度中等且小的克隆 (clone)，不透明度低的或过大的删除 (prune)。每 3k 步重置不透明度。

第10步 — 保存 checkpoint：保存 .ply (仅高斯参数) 和 .pth (完整 optimizer 状态)，可用于恢复训练或测试。

def render_set(model_path, name, iteration,
                 views, gaussians, pipeline,
                 background, train_test_exp, separate_sh):

        rendering = render(view, gaussians, pipeline,
                         background)["render"]
        torchvision.utils.save_image(

        torchvision.utils.save_image(
            view.original_image, f"{idx:05d}.png")

def render_sets(dataset, iteration, pipeline,
                    skip_train, skip_test, separate_sh):
    gaussians = GaussianModel(dataset.sh_degree)
    scene = Scene(dataset, gaussians,
                    load_iteration=iteration,
                    shuffle=False)
    if not skip_train:
        render_set(...scene.getTrainCameras()...)
    if not skip_test:
        render_set(...scene.getTestCameras()...)

scene/gaussian_model.py — 3D 高斯模型 ★核心类★

这是整个代码库最重要的文件。 GaussianModel 类管理所有 3D 高斯的参数、优化器、densification/pruning 逻辑。 每个高斯 = 完整协方差矩阵 Σ(S,R) + 位置 μ + 不透明度 α + 球谐系数 (视角相关颜色)。共 59 维参数。

从 COLMAP 稀疏点云初始化：位置=点云坐标，颜色=RGB→SH(DC)，scale=最近邻距离的 log，rotation=单位四元数，opacity=0.1。
优化 training_setup()
创建 Adam optimizer（6个参数组各不同lr：xyz, f_dc, f_rest, opacity, scaling, rotation），指数衰减调度器。
增删 densify_and_prune()
核心致密化：view-space 梯度大→clone(小高斯)或split(大高斯)，不透明度低/过大→prune。
IO save_ply() / load_ply()
序列化到 .ply（每个高斯一行：x,y,z,f_dc_*,f_rest_*,opacity,scale_*,rot_*），便于查看和交换。
# ═══ 激活函数 — 将 raw 参数映射到有效范围 ═══
def setup_functions(self):
    self.scaling_activation   = torch.exp       # log→正实数
    self.opacity_activation   = torch.sigmoid   # logit→[0,1]
    self.rotation_activation  = F.normalize     # 任意→单位四元数

    # 协方差构造: Σ = RSSᵀRᵀ
    def build_covariance_from_scaling_rotation(scaling, scaling_mod, rotation):
        L = build_scaling_rotation(scaling_mod * scaling, rotation)
        #   L = R · diag(s_mod * s)  →  Σ = L @ Lᵀ
        actual_covariance = L @ L.transpose(1, 2)
        return strip_symmetric(actual_covariance)
        #   只保留上三角6个分量 (协方差对称)
    self.covariance_activation = build_covariance_from_scaling_rotation

# ═══ 核心属性 (raw 参数 + property getter) ═══
class GaussianModel:
    self._xyz           # (N,3) — 世界坐标位置 μ
    self._features_dc   # (N,1,3) — SH 0阶 DC (基础色)
    self._features_rest # (N,15,3) — SH 1~3阶 (视角相关)
    self._scaling       # (N,3) — log(s) 尺度
    self._rotation      # (N,4) — 四元数 r
    self._opacity       # (N,1) — logit(α) 不透明度

    self.max_radii2D        # 屏幕空间最大半径 (prune用)
    self.xyz_gradient_accum # view-space 梯度累积 (densify用)
    self.denom              # 累积次数 (用于梯度归一化)

@property
def get_scaling(self):  return torch.exp(self._scaling)
@property
def get_opacity(self):  return torch.sigmoid(self._opacity)
@property
def get_rotation(self): return F.normalize(self._rotation)

高斯参数化 (Parameterization)

每个 3D 高斯在空间中定义为：

\[G(x) = \exp\left(-\frac{1}{2}(x-\mu)^T \Sigma^{-1}(x-\mu)\right)\]

其中协方差矩阵 \(\Sigma = RSS^T R^T\)：

  • \(S\) = \(\text{diag}(s_1, s_2, s_3)\) — 三轴尺度（存储为 \(\log(s)\) 以保证 >0）
  • \(R\) — 3×3 旋转矩阵（由四元数 \(r \in \mathbb{R}^4\) 构建，\(|r|=1\)）
  • 这种分解保证 \(\Sigma\) 始终是正定矩阵

为什么存 raw + 激活函数？优化需要无约束空间。scaling 存 log(s)（exp后 >0），opacity 存 logit(α)（sigmoid后 ∈[0,1]），rotation 用四元数（normalize后为单位四元数）。

球谐系数 (SH)：max 4 阶 = 16 系数 × RGB 3 通道 = 48 维（+ DC 3 维 = 51 维）。SH 让高斯颜色随观察方向变化，产生镜面反射等视角相关效果。

# ═══ 从点云初始化 ═══
def create_from_pcd(self, pcd, cam_infos, spatial_lr_scale):
    # 1. 位置 = 点云坐标
    points = torch.tensor(pcd.points).cuda()

    # 2. RGB → SH DC 分量 (0阶)
    color_SH = RGB2SH(torch.tensor(pcd.colors).cuda())
    features = zeros(N, 3, (max_sh_degree+1)**2)
    features[:, :3, 0] = color_SH     # DC
    features[:, 3:, 1:] = 0.0           # 高阶初始为0

    # 3. scale = 最近邻距离 (CUDA KNN)
    dist2 = distCUDA2(points)
    scales = torch.log(torch.sqrt(dist2)).repeat(1, 3)

    # 4. rotation = 单位四元数 (w=1, 无旋转)
    rots = zeros(N, 4); rots[:, 0] = 1

    # 5. opacity = sigmoid⁻¹(0.1) ≈ -2.2
    opacities = inverse_sigmoid(0.1 * ones(N, 1))

    # 6. 全部注册为 nn.Parameter
    self._xyz = nn.Parameter(points)
    self._features_dc  = nn.Parameter(features[:,:,0:1])
    self._features_rest = nn.Parameter(features[:,:,1:])
    self._scaling  = nn.Parameter(scales)
    self._rotation = nn.Parameter(rots)
    self._opacity  = nn.Parameter(opacities)

create_from_pcd() — 从 SfM 点云初始化

  • 位置直接从 COLMAP SfM 输出的稀疏点云获取（通常几千到几万点）
  • 颜色从点云 RGB 转为 SH DC (0阶) 分量——初始状态与视角无关
  • 尺度取每个点到最近邻的欧氏距离——稀疏区域高斯大，密集区域高斯小
  • 旋转初始化为无旋转（单位四元数 w=1）
  • 不透明度初始化为 0.1（中等偏低的合理初值）

所有参数注册为 nn.Parameter，PyTorch 自动追踪梯度。

# ═══ 训练配置 — 6个参数组，各自独立学习率 ═══
def training_setup(self, training_args):
    l = [
      {'params': [self._xyz],
       'lr': position_lr_init * spatial_lr_scale,  # ~1.6e-4
       "name": "xyz"},
      {'params': [self._features_dc],
       'lr': 0.0025, "name": "f_dc"},
      {'params': [self._features_rest],
       'lr': 0.0025 / 20.0, "name": "f_rest"},
      {'params': [self._opacity],
       'lr': 0.025, "name": "opacity"},
      {'params': [self._scaling],
       'lr': 0.005, "name": "scaling"},
      {'params': [self._rotation],
       'lr': 0.001, "name": "rotation"}
    ]
    self.optimizer = torch.optim.Adam(l, lr=0.0, eps=1e-15)
    # xyz 学习率按指数衰减
    self.xyz_scheduler_args = get_expon_lr_func(
        lr_init=position_lr_init * spatial_lr_scale,
        lr_final=position_lr_final * spatial_lr_scale, ...)

# ═══ 学习率调度 + SH递增 ═══
def update_learning_rate(self, iteration):
    for pg in self.optimizer.param_groups:
        if pg["name"] == "xyz":
            pg['lr'] = self.xyz_scheduler_args(iteration)

def oneupSHdegree(self):
    if self.active_sh_degree < self.max_sh_degree:
        self.active_sh_degree += 1

training_setup() — 优化器 & 调度策略

差异化学习率（不同物理含义的参数需要不同 lr）：

  • xyz 位置 — lr 最高，指数衰减（如 1.6e-4 → 1.6e-6）；乘以 spatial_lr_scale 适应不同场景尺度
  • 颜色特征 (SH) — DC lr=0.0025，高阶 lr 低 20 倍 (0.000125)，防止高频颜色过早拟合
  • 不透明度 — lr=0.025，高 lr 快速学习可见性
  • 尺度 + 旋转 — 中等 lr

SH 递增训练：每 1000 步提升一阶。训练初期只有 DC（视角无关色），几何收敛后逐步引入高阶 SH 捕捉视角相关效果。避免早期用高阶 SH 拟合噪声。

# ═══ 致密化 — Clone / Split / Prune ═══
def densify_and_prune(self, max_grad, min_opacity,
                       extent, max_screen_size, radii):
    grads = self.xyz_gradient_accum / self.denom
    self.tmp_radii = radii

    # Clone: 梯度大 + 尺度小 → 复制 (填充欠重建区)
    self.densify_and_clone(grads, max_grad, extent)

    # Split: 梯度大 + 尺度大 → 分裂 (细化过粗高斯)
    self.densify_and_split(grads, max_grad, extent)

    # Prune: α < 0.005 / 屏幕半径过大 / 世界尺度过大
    prune_mask = (self.get_opacity < min_opacity).squeeze()
    if max_screen_size:
        prune_mask |= (self.max_radii2D > max_screen_size)
        prune_mask |= (max(scaling) > 0.1 * extent)
    self.prune_points(prune_mask)

def densify_and_split(self, grads, grad_threshold, extent, N=2):
    selected = (grads >= grad_threshold)
             & (max(scaling) > percent_dense * extent)
    # 在高斯局部坐标系中采样 N 个新位置
    stds = self.get_scaling[selected].repeat(N, 1)
    samples = torch.normal(mean=0, std=stds)
    rots = build_rotation(self._rotation[selected]).repeat(N, 1, 1)
    new_xyz = rots @ samples + self.get_xyz[selected].repeat(N, 1)
    # 新尺度缩小
    new_scaling = scaling_inv(self.get_scaling[selected].repeat(N, 1) / (0.8*N))
    # 添加新点，删除原高斯
    self.densification_postfix(new_xyz, ..., new_scaling, ...)
    self.prune_points(selected)

def add_densification_stats(self, viewspace_point_tensor, update_filter):
    self.xyz_gradient_accum[update_filter] += \
        torch.norm(viewspace_point_tensor.grad[update_filter, :2], dim=-1, keepdim=True)
    self.denom[update_filter] += 1

致密化 (Densification) — 自适应控制高斯数量

这是 3DGS 最重要的贡献之一。训练过程中高斯从几千个动态增长到数百万个。

梯度累积：每步 add_densification_stats() 累积 view-space 位置梯度的 L2 范数。梯度大 = 该区域"欠重建"。

Clone（克隆）：梯度大 + 尺度小 → 需要更多高斯覆盖 → 复制一份让它们各自移动去覆盖不同区域。用于重建精细几何。

Split（分裂）：梯度大 + 尺度大 → 高斯太大覆盖了不该覆盖的区域 → 分裂成 N=2 个小高斯，在原高斯局部坐标系中采样新位置，尺度缩小为 1/(0.8·N)。

Prune（剪枝）：α<0.005（几乎透明）→删。屏幕半径>阈值（过大）→删。世界尺度>0.1·extent（可能是漂浮物）→删。

Opacity Reset：每 3000 步将 α clamp 到 ≥0.01，防止不透明度在训练早期被压低到难以恢复。

Densification 仅在前 15k 步执行（densify_until_iter），每 100 步一次（densification_interval），从第 500 步开始（densify_from_iter）。

Renderer 是连接高斯模型与 2D 图像的桥梁。 将 3D 高斯投影→排序→α-blend 合成像素颜色。 这个 Python 文件是 CUDA rasterizer 的 wrapper，真正的计算在 diff-gaussian-rasterization 子模块的 CUDA kernel 中。 
def render(viewpoint_camera, pc: GaussianModel, pipe,
            bg_color, scaling_modifier=1.0,
            separate_sh=False, override_color=None,
            use_trained_exp=False):

    # Step 1: 创建 screen-space points (梯度回传占位)
    screenspace_points = torch.zeros_like(
        pc.get_xyz, requires_grad=True) + 0
    screenspace_points.retain_grad()

    # Step 2: 配置光栅化参数
    raster_settings = GaussianRasterizationSettings(
        image_height, image_width,
        tanfovx, tanfovy,           # 视场角
        bg=bg_color,                # 背景色
        viewmatrix=world_view,       # 世界→相机 (4×4)
        projmatrix=full_proj,       # 世界→NDC (4×4)
        sh_degree=pc.active_sh_degree,
        campos=camera_center,       # 相机世界坐标
        prefiltered=False,
        debug=pipe.debug,
        antialiasing=pipe.antialiasing
    )
    rasterizer = GaussianRasterizer(raster_settings)

    # Step 3: 提取高斯属性
    means3D = pc.get_xyz
    opacity = pc.get_opacity
    scales = pc.get_scaling       # 或 precomputed cov3D
    rotations = pc.get_rotation

    # Step 4: SH → RGB (Python端可选)
    if pipe.convert_SHs_python:
        dir_pp = normalize(means3D - camera_center)
        sh2rgb = eval_sh(pc.active_sh_degree, shs_view, dir_pp)
        colors_precomp = clamp_min(sh2rgb + 0.5, 0.0)
    else:
        shs = pc.get_features  # CUDA端计算

    # Step 5: 调用 CUDA Rasterizer
    rendered_image, radii, depth_image = rasterizer(
        means3D, means2D,
        shs=shs, colors_precomp=colors_precomp,
        opacities=opacity,
        scales=scales, rotations=rotations,
        cov3D_precomp=None)

    # Step 6: 可选的 exposure 补偿
    if use_trained_exp:
        exposure = pc.get_exposure_from_name(
            viewpoint_camera.image_name)
        rendered_image = exposure @ rendered_image

    return {
        "render": rendered_image.clamp(0,1),
        "viewspace_points": screenspace_points,
        "visibility_filter": (radii > 0).nonzero(),
        "radii": radii,
        "depth": depth_image,
    }

render() — 渲染管线完整流程

  1. Screen-space points — 与高斯数量相同的 tensor，作为 2D 投影"占位符"。它的 .grad 就是 d(loss)/d(2D_position)，用于 densification 的梯度累积
  2. 光栅化设置 — 传入相机内外参、FOV、背景色、SH 阶数
  3. 提取参数 — 从 GaussianModel 获取所有高斯属性（经激活函数转换到有效范围）
  4. SH→RGB — Python端（convert_SHs_python=True）或 CUDA 端。核心是 eval_sh()，根据观察方向评估球谐函数
  5. CUDA Rasterizer — 核心计算：投影 3D→2D → tile排序 → α-blending → 输出图像（完全在 CUDA kernel 中）
  6. Exposure 补偿 — 可选，处理图像曝光不一致（Mip-NeRF 360 户外场景）

关键返回值：

  • render — RGB 图像 [3, H, W]
  • viewspace_points — 2D 投影点，.grad 用于 densification
  • visibility_filter — 实际参与渲染的高斯索引
  • radii — 每个高斯在屏幕上的半径（像素）
  • depth — 深度图（用于深度正则化）
# ═══ 渲染中的数学 ═══

# 1. 3D 高斯 → 2D 投影
#   Σ̂ = J W Σ Wᵀ Jᵀ
#   J: 投影变换的 Jacobian (仿射近似)
#   W: 世界→相机 view transform

# 2. 像素颜色 α-blending (按深度排序)
#   C = Σᵢ cᵢ · αᵢ · Πⱼ₌₁ⁱ⁻¹ (1 - αⱼ)
#   αᵢ = σᵢ · Ĝᵢ(pixel)
#   其中 Ĝᵢ 是投影后的 2D 高斯

# 3. Tile-based 排序
#   屏幕分 16×16 tile
#   每 tile 独立排序 (基数排序 O(n))
#   实现高度 GPU 并行化

可微分渲染的数学原理

投影：3D 协方差 \(\Sigma\) 投影为 2D 协方差 \(\hat{\Sigma} = J W \Sigma W^T J^T\)。\(J\) 是投影变换的仿射近似 Jacobian（在 3D 高斯中心处线性化）。

α-Blending：按深度排序（近→远），每个 pixel 的最终颜色 = 所有覆盖该 pixel 的高斯颜色的加权和，权重 = α × 前面所有高斯的累积透射率。当透射率 T < ε 时早停。

Tile-based 加速：屏幕分 16×16 tile，每个 tile 独立收集→排序→混合。这是 3DGS 能实时渲染的核心原因——高度并行 + 早停。

C0 = 0.28209479177387814
C1 = 0.4886025119029199
C2 = [1.0925484, -1.0925484, 0.31539156, -1.0925484, 0.5462742]
...

def eval_sh(deg, sh, dirs):
    """在单位方向 dirs 上评估 SH 基函数
    sh:  系数 [..., C, (deg+1)²]
    dirs: 单位方向 [..., 3]
    返回: RGB 颜色 [..., C]
    支持 deg 0~4
    """
    result = C0 * sh[..., 0]
    if deg > 0:
        x, y, z = dirs[..., 0], dirs[..., 1], dirs[..., 2]
        result = (result
            - C1*y * sh[..., 1]
            + C1*z * sh[..., 2]
            - C1*x * sh[..., 3])
        if deg > 1:
            xx, yy, zz = x*x, y*y, z*z
            xy, yz, xz = x*y, y*z, x*z
            result = (result
                + C2[0]*xy * sh[..., 4]
                + C2[1]*yz * sh[..., 5]
                + C2[2]*(2*zz-xx-yy) * sh[..., 6]
                + C2[3]*xz * sh[..., 7]
                + C2[4]*(xx-yy) * sh[..., 8])
        if deg > 2:
            ...  # deg=3,4 更高阶项
    return result

def RGB2SH(rgb):
    return (rgb - 0.5) / C0   # RGB[0,1] → SH DC

球谐函数 — 视角相关颜色编码

SH 是球面上的正交基函数。3DGS 用它们表示每个高斯在不同观察方向上的颜色：

\[\text{color}(\mathbf{d}) = \sum_{i=0}^{15} \text{SH}_i(\mathbf{d}) \cdot \mathbf{c}_i\]

每个高斯存储 SH 系数 \(c_i\)（不是直接颜色值）。渲染时根据观察方向 \(\mathbf{d}\)（高斯→相机）评估 SH 基函数，加权求和得 RGB。

RGB2SH() 将 [0,1] RGB 转为 SH 0阶 (DC) 系数，用于从点云颜色初始化。

3DGS 支持到 4 阶 (deg=3)，即 16 系数 × 3 通道 = 每个高斯 48 维额外颜色参数（加上 DC 的 3 维共 51 维颜色）。

实现来自 PlenOctrees 项目，用硬编码的 SH 多项式系数，比动态计算快得多。

def l1_loss(network_output, gt):

def ssim(img1, img2, window_size=11, size_average=True):
    channel = img1.size(-3)
    window = create_window(window_size, channel)  # 11×11高斯核

    # 滑动窗口计算统计量
    mu1 = F.conv2d(img1, window, padding=window_size//2, groups=channel)
    mu2 = F.conv2d(img2, window, padding=window_size//2, groups=channel)
    sigma1_sq = F.conv2d(img1*img1, window, ...) - mu1.pow(2)
    sigma2_sq = F.conv2d(img2*img2, window, ...) - mu2.pow(2)
    sigma12   = F.conv2d(img1*img2, window, ...) - mu1*mu2

    C1, C2 = 0.01**2, 0.03**2
    ssim_map = ((2*mu1*mu2 + C1) * (2*sigma12 + C2)) / \
              ((mu1**2+mu2**2+C1) * (sigma1_sq+sigma2_sq+C2))
    return ssim_map.mean()

def fast_ssim(img1, img2):
    return FusedSSIMMap.apply(C1, C2, img1, img2).mean()
    #   CUDA 融合 forward+backward

Loss = L1 + λ·(1-SSIM)

  • L1 Loss — 逐像素绝对误差，比 L2 更鲁棒，保留高频细节
  • SSIM — 结构相似性指标，由亮度/对比度/结构三个分量组成。11×11 高斯滑动窗口。值域 [0,1]，1=完全相同
  • 组合公式：loss = (1-λ)·L1 + λ·(1-SSIM)，λ=0.2（默认）
  • fast_ssim (FusedSSIMMap) — 自定义 PyTorch autograd Function：forward 调用 CUDA kernel，backward 也有对应 CUDA kernel，避免 Python 端多次 GPU→CPU 传输

def inverse_sigmoid(x):
    return torch.log(x / (1 - x))

# 指数衰减 LR (来自 Plenoxels)
def get_expon_lr_func(lr_init, lr_final,
        lr_delay_steps=0, lr_delay_mult=1.0, max_steps=1e6):
    def helper(step):
        if lr_delay_steps > 0:
            delay = lr_delay_mult + (1-lr_delay_mult) * \
                    sin(π/2 * step/lr_delay_steps)
        else: delay = 1.0
        t = step / max_steps
        log_lerp = exp(log(lr_init)*(1-t) + log(lr_final)*t)
        return delay * log_lerp
    return helper

# World→Camera 变换 (支持 NERF 归一化)
def getWorld2View2(R, T, translate, scale):
    Rt = [[Rᵀ, T], [0, 1]]
    C2W = inv(Rt)
    cam_center = (C2W[:3,3] + translate) * scale
    C2W[:3,3] = cam_center
    return inv(C2W)

# OpenGL 风格透视投影矩阵
def getProjectionMatrix(znear, zfar, fovX, fovY):
    P = zeros(4,4)
    P[0,0] = 2*znear / (right-left)
    P[1,1] = 2*znear / (top-bottom)
    P[2,2] = zfar/(zfar-znear)
    P[2,3] = -zfar*znear/(zfar-znear)
    P[3,2] = 1
    return P

# FoV ↔ 焦距 互转
def fov2focal(fov, pixels):
    return pixels / (2*tan(fov/2))
def focal2fov(focal, pixels):
    return 2*atan(pixels/(2*focal))

        img1.shape[0], -1).mean(1, keepdim=True)

参数系统 — 声明式元编程

ParamGroup 是一个精巧的设计：将 Python 类属性自动注册为 argparse 参数。属性名即 --name 参数名，_前缀的属性同时获得单字母短参数（如 _source_path→-s）。

推理时 get_combined_args() 合并命令行参数和训练时保存的 cfg_args，确保推理参数与训练一致。

关键超参说明：

  • lambda_dssim=0.2 → loss = 0.8·L1 + 0.2·(1-SSIM)
  • densify_until_iter=15000 → 后15k步不发新高斯，纯调参数
  • percent_dense=0.01 → scale > 场景半径1%的高斯 split，否则 clone
  • densify_grad_threshold=0.0002 → view-space 梯度阈值
  • random_background=False → 开启后每步用随机背景色训练（提升泛化）

C++/CUDA 子模块 — 底层加速

Python 只是前端，真正的计算在 CUDA kernel 中完成。 三个子模块（submodules/）提供了核心性能关键操作。
光栅化 diff-gaussian-rasterization
核心可微分渲染器。 forward: 投影+排序+光栅化+α-blend；backward: 梯度传播。包含 tile-based 排序、各向异性高斯评估等 CUDA kernel。
关键类: GaussianRasterizer, GaussianRasterizationSettings, SparseGaussianAdam
KNN simple-knn
CUDA K-近邻距离计算 distCUDA2()。从点云计算每个点到最近邻的距离，用于初始化高斯尺度。
SSIM fused-ssim
融合的 CUDA SSIM：forward+backward 都在 GPU 上完成，提供 fused_ssim() 函数。
优化器 SparseGaussianAdam
稀疏 Adam（在 rasterizer 中）：只更新 visible (radii>0) 的高斯，跳过被遮挡/视锥外的，显著加速训练。
// ── diff-gaussian-rasterization 内部 CUDA kernel: ──

// 1. preprocess: 投影 3D → 2D
//    Σ̂ = J W Σ Wᵀ Jᵀ
//    提取 2D 均值 + 协方差逆
//    视锥剔除

// 2. duplicateWithKeys + sort:
//    为每个 tile (16×16) 收集覆盖它的高斯
//    基数排序按深度排序 (O(n))

// 3. render (核心 α-blending):
//    for each pixel in tile:
//      T = 1.0  // 累积透射率
//      for each gaussian in tile (depth sorted):
//        weight = α * G_2D(pixel)
//        color += weight * c * T
//        T *= (1 - weight)
//        if T < ε: break  // 早停

// 4. backward: 梯度反向传播
//    dL/dμ̂ → dL/dμ (3D位置)
//    dL/dΣ̂ → dL/dS (scale) + dL/dR (rotation)
//    dL/dcolor, dL/dα 等

CUDA 光栅化 — 为什么这么快？

Tile-based 架构：屏幕分 16×16 tile，每个 tile block 由一个 CUDA thread block 处理。每个 tile 维护自己的高斯列表和深度排序，block 之间完全独立并行。

基数排序 O(n)：用 32-bit 整数 key（深度 quantized）做基数排序，仅需 4 pass（每 8 bit 一次），比比较排序的 O(n log n) 快得多。

早停 (Early Termination)：当累积透射率 T < ε 时停止遍历。在遮挡严重的区域，大部分高斯被前面的不透明高斯挡住，早停大幅减少计算。

Forward/Backward 一体：forward 时同时缓存必要中间结果（如每个 pixel 的最后一个高斯索引），backward 直接复用，避免重复计算。

SparseGaussianAdam：仅对 radii>0（实际参与渲染）的高斯执行 optimizer step。训练初期大量高斯不可见时加速显著。

PyTorch 绑定在 submodules/diff-gaussian-rasterization/diff_gaussian_rasterization/__init__.py 中，通过 torch.utils.cpp_extension.load 动态加载编译好的 CUDA 扩展。
